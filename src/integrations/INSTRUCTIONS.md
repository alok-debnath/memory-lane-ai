# Integrations — INSTRUCTIONS.md

## Purpose

Third-party service integrations. Currently contains the Supabase client and auto-generated types.

## Do

- Import the Supabase client from `@/integrations/supabase/client`.
- Use auto-generated types from `@/integrations/supabase/types.ts` for type safety.

## Don't

- **NEVER** edit `client.ts` — it is auto-generated by Lovable Cloud.
- **NEVER** edit `types.ts` — it is auto-generated from the database schema.
- **NEVER** create additional Supabase client instances.
- **NEVER** hardcode Supabase URLs or keys — use environment variables.

## Conventions

- If adding a new integration, create a subfolder (e.g., `integrations/stripe/`).
- Each integration should export a single client/instance.
